SCALe2 : DB Design for per-project SQLite files in backup

Created by Wiki Administrator, last modified by David Svoboda on Jan 12, 2018

When a new SCALe project is created using the SCALe GUI, a new SQLite-formatted file (each named "db.sqlite") is created in a folder named with an integer (increases by 1 with each new project), in the scale.app/archive/backup directory. E.g,:

  • $SCALE_HOME/scale.app/archive/backup/1/db.sqlite
  • $SCALE_HOME/scale.app/archive/backup/2/db.sqlite

Since every project has a unique ID, below we use the $PROJECT variable to represent a particular project ID, eg:

  • $SCALE_HOME/scale.app/archive/backup/$PROJECT/db.sqlite

The internal format of these database files is the same as for DB Design for external.sqlite3.

Creating a new project

This process is also done when a project is edited:

update db/development.sqlite3 -> db/external.sqlite3
copy db/external.sqlite3 -> db/backup/$PROJECT/external-`timestamp`.sqlite3
load SA-output -> archive/backup/$PROJECT/db.sqlite
copy archive/backup/$PROJECT/db.sqlite -> db/external.sqlite3
copy db/external.sqlite3 -> db/backup/$PROJECT/external.sqlite3

Uploading a new database

This is done when the user selects "Create DB".

upload -> archive/backup/$PROJECT/db.sqlite
copy archive/backup/$PROJECT/db.sqlite -> db/external.sqlite3
update db/external.sqlite3 -> db/development.sqlite3
copy db/external.sqlite3 -> db/backup/$PROJECT/db.sqlite

Exporting a database

This is done when the user selects 'Export DB'.

copy db/backup/$PROJECT/external.sqlite3 -> db/external.sqlite3
update db/development.sqlite3 -> db/external.sqlite3
download db/external.sqlite3
