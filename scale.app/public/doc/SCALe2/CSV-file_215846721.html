SCALe2 : CSV file

Created by Wiki Administrator on Dec 04, 2017

You can extract data from an SQL database into comma-separated-values (CSV), which is useful for importing into Excel or any other spreadsheet application. To do this, you will use scale2csv.py, which is located in the $SCALE_HOME/scale.app/scripts directory of the SCALe web app.

scale2csv.py <db> --constraint <select-arg>? --link excel --prefix <prefix> > <csv-file>

You'll have to provide selection arguments, of course.

You can also insert links to a Web server (or the local filesystem) into this table. You will use GNU Global to build web pages out of a source tree. To do this, go to your source directory and type

htags -aghnosTx --show-position

This creates an HTML subdirectory with the web pages. (You could now remove the source code; the HTML pages are independent of the source code.)

You'll also need to provide a <prefix> argument to scale2csv.py. This is a URL each link in the spreadsheet will point to. It directs your web browser to your source directory. The links all then point inside the HTML subdirectory in your source directory.

To update your database with modified data, you can use this command.

./csv2scale.py < <csv-file> | sqlite3 <db>

It constructs several SQL UPDATE statements, which updates the database with your new verdicts and flags (from the CSV file). Any other fields are ignored.

Occasionally, you will encounter a diagnostic with multiple messages associated with it, but the table you get shows only one 'primary' message. This command produces a CSV file consisting of all the messages associated with a single diagnostic.

./msg.py --link excel --prefix <prefix> <db> <diagnostic-id> > <csv-file>

This is useful for diagnostics that have multiple messages. Whereas the other routines only produce one message per diagnostic (said message also containing a filename and line number), this routine produces all the messages, producing a CSV table of all the messages. The diagnostic ID it requires is a number which is available in the CSV files produced by scale2csv.py.
