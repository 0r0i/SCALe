SCALe2 : Export Analysis Results from the Web Application (Internal)

Created by Wiki Administrator, last modified by Lori Flynn on Jan 16, 2018

Prerequisites for This Step

  1. Access to a SCALe web app (you need the URL of the application plus a username and password, obtainable from your SCALe administrator)
  2. A supported web browser (currently Chrome, Firefox, or Internet Explorer)
  3. The dos2unix diagnostics from Coverity and Fortify uploaded to the web application (as done in Step 3)

After auditing diagnostics in the SCALe web app, you can export the results of your audit to a SQLite or CSV file. Note that a SQLite export contains all the data from the project, but a CSV export does not (e.g., a CSV export does not contain secondary diagnostic messages, may not contain all (or any) of the primary diagnostic messages, and may not include diagnostic IDs). An exported SQLite or CSV file is often further analyzed to produce statistics for a report.

To obtain a CSV file, go to the SCALe web app homepage and click the Export CSV link for the desired project.

In this case, we download a CSV file for the dos2unix audit.

Save the file to your local system. You can now open this file in a spreadsheet tool such as Excel. The CSV file contains information about every diagnostic in the project: the tool that produced the diagnostic, the file and line number where the diagnostic occurs, the message from the diagnostic, and so on. (However, as mentioned above, a SQLite export contains all the data from a project but a CSV export does not.) See this table for a detailed explanation of all the diagnostic fields.
