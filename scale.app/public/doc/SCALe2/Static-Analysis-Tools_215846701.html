SCALe2 : Static Analysis Tools

Created by Wiki Administrator, last modified by Lori Flynn on Jan 19, 2018

When auditing a codebase, you should use all the tools (listed below) that you possess and that are supported by your operating system. You may use tools on multiple operating systems if the code you are testing is portable. For example, if you have a C codebase that compiles under both Windows and Linux, you may use tools listed under both platforms.

Each platform supports multiple tools. For each flaw-finding tool, there is a section that instructs precisely how to run the tool. These instructions should produce a file containing the tool's diagnostics (for general flaw-finding tools). An output file containing code metrics is required for using output from the code metrics tools.

In the following pages, we provide a number of commands to use for in the flaw-finding tools for Windows and Linux platforms. We assume that the $SCALE_HOME environment variable refers to the path where the SCALe app is installed. Tools that are installed in nonstandard locations occasionally require commands that include their path. We assume you have environment variables specifying their install path in these commands. For example, Fortify comes with a program called ReportGenerator, and so we assume $FORTIFY_HOME is set so that you can invoke $FORTIFY_HOME/bin/ReportGenerator on Linux.

The tools and platforms integrated into SCALe are as follows:

Linux + GCC

These flaw-finding tools can be used on both C and C++ programs that are built using the GNU Compiler Collection on Linux:

  • GCC warnings
  • Rosecheckers
  • Coverity Prevent
  • Fortify
  • Cppcheck

These code metrics tools can be used:

  • CCSM
  • Lizard (Lizard can be used on C, C++14, C#, Java, Python, JavaScript, etc.)

Windows + MSVC

These flaw-finding tools can be used on both C and C++ programs that are built using Microsoft Visual Studio on Windows:

  • MSVC's Static Analyzer
  • PC-Lint
  • Fortify
  • Coverity Prevent
  • LDRA
  • Cppcheck

These code metrics tools can be used:

  •  Understand (Understand can be used on C/C++, C#, Java, Python, ADA etc. )  

Java

These flaw-finding tools can be used for Java programs. We assume they can be built using Oracle's JDK on Linux:

  • Eclipse
  • FindBugs
  • Fortify
  • Coverity Prevent

These code metrics tools can be used:

  • Lizard (Lizard can be used on C, C++14, C#, Java, Python, JavaScript, etc.)
  • Understand (Understand can be used on C/C++, C#, Java, Python, ADA etc. )  

Perl

These flaw-finding tools operate on Perl source code. 

  • Perl Critic
  • B Lint

 Currently there are no Perl code metrics tools integrated with SCALe. In the future, these code metrics tools could potentially be added:

  • Perl Metrics Simple
  • Perl Metrics Lite
