SCALe2 : MSVC's Static Analyzer

Created by Wiki Administrator on Dec 04, 2017

Microsoft Visual Studio comes with a static analyzer for C/C++ code.  MSVC's different categories of diagnostics are documented   on MSDN.

To harvest the warnings from MSVC's static analyzer on a project (that already builds correctly under MSVC), first import a .sln (solution) file into MSVC. Then select Build → Run Code Analysis on Solution or press Alt-F11.

Formatting Output For SCALe

Once MSVC has finished its analysis, select all of its output, right-click, and select Copy. Then paste into Notepad (or any other text editor) and save the text output to a file.

CAUTION: This output provides warnings about a different set of code flaws than the output from running MSVC normally. (This output is not a superset!) To also obtain warnings for the MSVC compiler's (additional) checkers, output from running MSVC normally is also needed. SCALe includes mappings between MSVC compiler checkers and CERT rules (from MSVC compiler standard runs), as well as mappings between MSVC's Static Analyzer Tool and CERT rules.
