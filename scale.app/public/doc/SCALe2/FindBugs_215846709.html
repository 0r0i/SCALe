SCALe2 : FindBugs

Created by Wiki Administrator on Dec 04, 2017

FindBugs is an open-source static analysis tool.  FindBugs' different categories of diagnostics are documented  on FindBugs' website .

FindBugs only operates on Java .class files, so you must first build your codebase.

Command Line

The commands to run FindBugs are:

$FINDBUGS_HOME/bin/findbugs  -textui  -low  -progress  -xml:withMessages  -xargs < cp.txt  -output findbugs.xml

This produces an XML file of FindBugs diagnostics.

If FindBugs runs out of memory, you can instruct it to use more with these arguments: -jvmArgs "-Xmx2048m"

The cp.txt input file indicates the classpath (that is, Jars and source directories) that FindBugs should analyze. The input file should contain the absolute pathname of one jar file per line. These commands in a source directory might produce a sufficient cp.txt file:

find . -name classes -print > cp.txt
find . -name \*.jar -print >> cp.txt
find ~/.m2/repository -name \*.jar -print >> cp.txt

GUI

Once you have invoked the FindBugs GUI, select File->New Project.
Then provide locations for class files, auxiliary classes, and source files. For auxiliary classes, you can select multiple jars at once.Then let FindBugs do its analysis.

Maven

FindBugs is already integrated with Maven. To run FindBugs on a Maven project, you must  integrate FindBugs into the top-level pom.xml file. To do this, follow the use case scenario 3 in this internet tutorial , and build the project. The directory tree will become peppered with findbugsXml.xml files.
