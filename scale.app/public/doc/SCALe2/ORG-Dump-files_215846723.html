SCALe2 : ORG Dump files

Created by Wiki Administrator on Dec 04, 2017

SQL allows you to dump the entire contents of a database (not just a single table). The command

sqlite3 <db> .dump > <sql-file>

fills <sql-file> with a series of SQL instructions for re-creating the database from scratch. This command recreates a new database from a dump:

sqlite3 <db> < <sql-file>

While useful, the <sql-file> is not particularly pretty. An easier way to manipulate the data is to convert it into an ORG file. To do this

sqlite3 <db> .dump | ./dbdump2org.py > <org-file>

The dbdump2org.py program lives in the $SCALE_HOME/scale.app/scripts directory.

After making any desired changes to the ORG file, you can convert it back to an SQL database with this command:

./org2dbdump.py < <org-file> | sqlite3 <db>
